# English Anywhere Lab - PRD（v0.2）

## 1. 文档信息
- 文档类型：产品需求文档（PRD）
- 版本：v0.2
- 日期：2026-03-04
- 状态：Draft -> 可进入设计与开发拆分

## 2. 背景与问题定义

多数英语学习者存在以下问题：
- 学习方法单一，输入和输出脱节
- 资源分散，内容质量和难度不稳定
- 学习节奏难以持续，进度不可视化

产品目标是提供一个跨端统一、资源可持续、并可量化进步的学习系统。

## 3. 产品目标与非目标

### 3.1 目标（Goals）
- G1：建立可持续学习闭环（输入、复习、输出）
- G2：提供稳定资源来源 + AI 个性化生成能力
- G3：支持移动端和 PC 随时学习、同步进度
- G4：通过数据看板帮助用户了解学习状态与提升趋势

### 3.2 非目标（Non-Goals）
- N1：首版不做社交社区
- N2：首版不做真人直播教学
- N3：首版不做大而全考试题库

## 4. 用户画像

### Persona A：碎片化时间学习者
- 特征：工作日忙，单次学习时间 10-20 分钟
- 诉求：快速进入学习、自动安排内容、不中断

### Persona B：场景导向学习者
- 特征：为工作或旅行等具体场景学习
- 诉求：希望内容贴近真实语境，能学以致用

### Persona C：重启学习者
- 特征：曾学习但长期断档
- 诉求：需要低门槛恢复节奏和正反馈

## 5. 关键场景与用户故事

1. 新用户完成水平设定后，系统自动生成一周学习计划。
2. 用户在通勤中打开手机完成 10 分钟快学。
3. 用户在晚上用 PC 做完整训练并查看周进度变化。
4. 用户想学“外贸英语”，系统生成对应领域学习包并加入计划。

## 6. 功能需求

### 6.1 Onboarding 与目标设定
- FR-001：支持邮箱/第三方登录
- FR-002：支持 CEFR 自评 + 快速分级测试
- FR-003：设置学习目标（领域、每日时长、每周学习天数）
- FR-004：完成设定后生成首周计划

验收标准：
- AC-001：新用户 3 分钟内可完成初始化
- AC-002：计划中至少包含新学、复习、输出三类任务

### 6.2 今日学习页（核心入口）
- FR-005：展示今日任务摘要（预计时长、任务数量、优先级）
- FR-006：支持“10 分钟快学”和“20 分钟完整模式”
- FR-007：支持断点续学（离开后可继续）

验收标准：
- AC-003：进入今日学习页到开始任务 <= 2 次点击
- AC-004：中断后重新进入可恢复到上次任务节点

### 6.3 学习包与内容中心
- FR-008：支持按难度、领域、技能筛选学习包
- FR-009：支持导入稳定资源来源（人工整理或系统抓取后清洗）
- FR-010：支持 AI 一键生成学习包（词汇、对话、练习、测验）

验收标准：
- AC-005：每个学习包必须有难度标签和预计时长
- AC-006：AI 生成失败可自动重试一次并返回错误原因

### 6.4 复习中心（SRS/FSRS）
- FR-011：卡片支持四级评分（Again/Hard/Good/Easy）
- FR-012：系统按到期时间自动调度复习队列
- FR-013：对高遗忘词提供强化（例句、图像、额外复习）

验收标准：
- AC-007：到期卡片可在 1 秒内加载
- AC-008：评分后下一次到期时间更新正确

### 6.5 输出训练
- FR-014：口语任务（跟读、复述、情景问答）至少支持其一
- FR-015：写作任务（句子改写、短段写作）至少支持其一
- FR-016：AI 提供反馈（准确性、流畅度、词汇建议）

验收标准：
- AC-009：每次学习至少有 1 个输出任务可选
- AC-010：反馈在 5 秒内返回（P95）

### 6.6 学习进度与分析
- FR-017：记录日学习时长、任务完成率、复习正确率
- FR-018：展示连续学习天数（streak）
- FR-019：展示周报/月报（提升点、薄弱点、下周建议）

验收标准：
- AC-011：看板数据按天聚合，刷新延迟 <= 5 分钟
- AC-012：支持按技能维度筛选（听/说/读/写）

### 6.7 跨端与可用性
- FR-020：PWA 支持移动端和桌面端适配
- FR-021：支持离线缓存近期学习任务
- FR-022：支持跨端账号同步和多设备接续学习

验收标准：
- AC-013：主流程在 iOS Safari / Android Chrome / 桌面 Chrome 可用
- AC-014：离线可完成最近 1 个学习包中的已缓存任务

### 6.8 核心模块功能阐述（MVP 可执行规格）
本节用于补齐“功能如何工作”的业务规格，作为 API 契约之外的产品行为基线。

#### 6.8.1 学习计划引擎（Plan Orchestrator）
功能目标：
- 将用户目标时长转成当天可执行任务序列，避免“只有资源没有编排”。

输入：
- 用户画像：`current_level`、`target_domain`、`daily_minutes`、`weekly_goal_days`
- 学习状态：到期复习数、未完成任务、最近学习表现
- 服务端时间：当日日期、周目标完成进度

输出：
- 今日计划（`daily_plan`）：任务列表、建议顺序、预计总时长
- 任务结构：`new / review / output` 三类任务

业务规则：
- 默认配比：新学 40% + 复习 40% + 输出 20%（按负载可调整）
- 若到期复习任务过多，优先保障复习（防止逾期堆积）
- 断点续学优先回到未完成任务，不重复发新任务

状态流转：
- 计划：`draft -> active -> completed | expired`
- 任务：`pending -> in_progress -> done | skipped`

异常与降级：
- 若计划计算失败，回退到模板计划（固定 10/20 分钟包），确保“可学”。

#### 6.8.2 学习包中心（Resource Pack）
功能目标：
- 提供可直接学习的结构化内容，兼容“稳定来源导入”和“AI 生成”两条链路。

输入：
- 筛选条件：领域、难度、技能、预计时长
- AI 生成参数：主题、CEFR 等级、卡片数量

输出：
- `resource_packs / lessons / cards / output_tasks` 结构化数据
- 生成任务状态：`queued/running/success/failed`

业务规则：
- 每个学习包必须具备 `level + domain + estimated_minutes`
- AI 生成结果必须通过：JSON 结构校验 + 业务 QC（重复度、字段完整性、难度匹配）
- 任务对象必须落库后才可被学习流程引用；禁止“随机生成且不落库”的临时任务进入正式任务流

异常与降级：
- 生成失败自动重试一次；仍失败返回明确错误码与可读原因
- 超过日额度直接拒绝，不入队

#### 6.8.3 复习中心（FSRS/SRS）
功能目标：
- 通过服务端调度控制复习节奏，稳定提升长期记忆保持率。

输入：
- `due_at <= server_now` 的卡片队列
- 用户评分：`again/hard/good/easy`

输出：
- 更新后的卡片记忆状态（`stability/difficulty/next_due_at`）
- 审计日志：`review_logs`

业务规则：
- 调度时间以服务端接收时间为准；客户端时间仅用于审计与离线排序
- 写入必须幂等（`idempotency_key`）
- 同一卡片并发提交时采用版本校验或事务保护，避免状态回退

状态流转：
- 卡片：`new -> learning -> review -> relearning`
- 复习记录：只追加，不覆盖历史

异常与降级：
- FSRS 计算失败时，回退到简化间隔策略并记录告警

#### 6.8.4 输出训练（Writing/Speaking）
功能目标：
- 提供“可提交答案 + 可获得反馈”的输出闭环，而非仅展示题目。

输入：
- 任务定义：题干、参考答案、难度、技能标签
- 用户提交：文本答案（MVP 必选），语音答案（可选）

输出：
- 结构化反馈：准确性、表达自然度、词汇建议、改写建议
- 分数区间：0-100（可映射等级）

业务规则：
- 反馈结构必须做严格校验：字段存在、类型正确、分数区间合法
- 任务来源必须为已落库的 `output_tasks`，禁止临时虚拟任务参与正式计分
- 提交与反馈状态应保持一致：若反馈失败，状态需显式标记为待重试或失败，不允许“返回错误但数据看似已成功”

状态流转：
- 任务：`assigned -> submitted -> evaluated -> closed`
- 提交：`received -> processing -> success | failed`

异常与降级：
- LLM 不可用时可返回“基础规则反馈”（模板化）并提示稍后重评

#### 6.8.5 学习进度与分析（Progress）
功能目标：
- 将原始学习行为转为可解释的趋势指标，支持自我反馈与目标调整。

输入：
- 事件流：学习开始、任务完成、复习提交、输出提交
- 聚合周期：日/周/月

输出：
- `progress_daily` 聚合指标
- 周报/月报摘要：提升点、薄弱项、建议动作

业务规则：
- 统计口径统一以服务端事件时间为准
- 指标按“听/说/读/写”可切片
- 聚合延迟目标 <= 5 分钟

异常与降级：
- 聚合任务失败可重跑；看板展示最后一次成功快照并标注时间

#### 6.8.6 跨端离线与同步（PWA Sync）
功能目标：
- 保证用户在弱网/离线情况下仍能学习，网络恢复后一致回放。

输入：
- 客户端离线事件队列（按顺序带 `event_id`）

输出：
- 服务器确认结果（已接收、冲突、重复）

业务规则：
- 同一事件幂等写入（重复上报不重复计数）
- 冲突处理优先“服务端时序 + 业务主键约束”
- 客户端仅可缓存最近学习包与必要任务数据

状态流转：
- 事件：`queued -> sent -> acked | conflicted`

异常与降级：
- 长时间离线导致资源过期时，允许“完成本地学习 + 同步时重映射资源”

#### 6.8.7 音频生成（TTS，支撑 AI 学习资源）
功能目标：
- 为 AI 生成的词卡/例句提供可听输入，提升听力与跟读训练可用性。

输入：
- 待配音文本、voice 参数、格式参数

输出：
- 对象存储中的音频文件 URL（回填 `cards.audio_url`）

业务规则：
- 纯 CPU、离线可运行
- 默认本地模型推理，异步任务生成，不阻塞主学习流程
- 使用确定性对象键去重，避免重复生成和重复存储
- 存储必须兼容 `local|s3` 配置切换

异常与降级：
- 生成失败进入可重试队列；超过重试上限标记失败并暴露诊断信息

## 7. 非功能需求

### 7.1 性能
- 首页首屏可交互时间（TTI）<= 3s（4G 网络）
- 关键接口 P95 响应 <= 800ms（不含 AI 生成接口）

### 7.2 可靠性
- 月度可用性 >= 99.5%
- 学习日志和复习日志必须幂等写入

### 7.3 安全与隐私
- 密码哈希存储，不明文落库
- 学习数据按用户隔离，默认不公开
- 支持用户导出和删除个人学习数据

## 8. 指标体系（MVP）

### 8.1 北极星指标
- 有效学习日（用户在一天内完成新学+复习+输出任一闭环）

### 8.2 关键指标
- D1/D7/D30 留存
- 周学习天数
- 复习正确率
- AI 学习包使用率
- 输出任务完成率

## 11. 决策落地（已定）
- D-001：首版输出反馈优先支持写作纠错与口语基础反馈，完整语音精评分后置到 V1.1。
- D-002：外部稳定资源默认仅存元数据与来源链接，不默认存全文。
- D-003：AI 生成按用户日额度限流（默认 2 次/天），超限拒绝并提示次日重试。
- D-004：数据库默认 SQLite（低成本），PostgreSQL 为可选配置。
- D-005：M1 接口单一真值采用 `doc/07-openapi-m1.yaml`。`doc/07-openapi.yaml` 作为后续阶段的全量路线图契约，不作为 M1 实现验收基线。
- D-006：复习调度时间以服务端接收时间为准；客户端 `reviewed_at` 仅用于审计和离线重放排序，不直接决定 `next_due_at`。
